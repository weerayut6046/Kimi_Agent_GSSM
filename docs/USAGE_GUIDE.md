## ภาพรวมระบบ

### 🔐 ระดับสิทธิ์ผู้ใช้ (User Roles)

| บทบาท | สิทธิ์การใช้งาน |
|-------|----------------|
| **Admin** | จัดการพนักงาน, ตั้งค่าระบบ, ดูรายงานทั้งหมด, จัดการกะ, จัดการสาขา |
| **Manager** | อนุมัติคำขอลา/สลับกะ, ดูรายงาน, จัดการตารางงาน, สลับสาขาได้ |
| **Staff** | ลงเวลาเข้า-ออกงาน, ยื่นคำขอลา, ยื่นคำขอสลับกะ, บันทึกบัญชี (เฉพาะสาขาตัวเอง) |

### 🏢 การแยกสาขา (Branch/Station)

ระบบรองรับการทำงานหลายสาขา:
- พนักงานแต่ละคนสังกัดสาขาเดียว (`stationId` ในข้อมูลพนักงาน)
- **Admin** สามารถเลือกดู "ทุกสาขา" หรือเลือกสาขาเฉพาะได้
- **Manager** สามารถสลับสาขาได้ (ถ้าดูแลหลายสาขา)
- **Staff** เห็นเฉพาะสาขาของตัวเอง ไม่สามารถเปลี่ยนสาขาได้

## การคำนวณบัญชีรายวัน

### 📊 โครงสร้างข้อมูลบัญชีรายวัน

```typescript
interface DailyAccounting {
  date: string;           // วันที่บันทึก
  shiftId: string;        // รหัสกะ
  employeeId: string;     // รหัสพนักงาน
  
  // 1. ยอดลิตรจากมิเตอร์ (Fuel Meter)
  fuelMeter: FuelMeterReading;
  
  // 2. ยอดขายน้ำมัน (ลิตร) - คำนวณอัตโนมัติ
  fuelSales: {
    '95': number;         // จำนวนลิตร Gas 95
    'B7': number;         // จำนวนลิตร B7
    'B10': number;        // จำนวนลิตร B10
    'Diesel': number;     // จำนวนลิตร Diesel
  };
  
  // 3. ยอดเงินจากการขายน้ำมัน - คำนวณอัตโนมัติ
  fuelAmount: {
    '95': number;
    'B7': number;
    'B10': number;
    'Diesel': number;
  };
  
  // 4. ยอดเงินรวมจากน้ำมัน (บาท)
  totalFuelAmount: number;
  
  // 5. ยอดเงินที่จอ (จากระบบคำนวณ)
  systemAmount: number;
  
  // 6. ยอดเงินสดที่นับได้ (คำนวณจากตู้จ่าย)
  cashAmount: number;
  
  // 7. ยอดเงินสดที่นับได้จริง (กรอกเอง)
  actualCashCounted: number;
  
  // 8. ยอดเงินในแต่ละตู้จ่าย
  dispenserCash: CashAmountReading;
  
  // 9. รายการอื่นๆ
  items: {
    twoT: number;         // 2T
    capital: number;      // เงินทุน
    transfer: number;     // เงินโอน
    others: number;       // อื่นๆ
  };
  
  // 10. สรุป
  totalAmount: number;    // ยอดสุทธิ
  difference: number;     // ยอดขาด/เกิน (+ = เกิน, - = ขาด)
}
```

## สูตรการคำนวณ

### 1️⃣ คำนวณยอดขายน้ำมัน (ลิตร)

```
ยอดขายต่อหัวจ่าย = max(0, มิเตอร์สิ้นสุด - มิเตอร์เริ่มต้น)

ตัวอย่าง:
D1N1 (ตู้ 1 หัว 1 - Gas 95) = max(0, มิเตอร์สิ้นสุด - มิเตอร์เริ่มต้น)
D1N2 (ตู้ 1 หัว 2 - Diesel) = max(0, มิเตอร์สิ้นสุด - มิเตอร์เริ่มต้น)
D2N1 (ตู้ 2 หัว 1 - Gas 95) = max(0, มิเตอร์สิ้นสุด - มิเตอร์เริ่มต้น)
D2N2 (ตู้ 2 หัว 2 - B7) = max(0, มิเตอร์สิ้นสุด - มิเตอร์เริ่มต้น)

รวมลิตรแต่ละประเภท:
Gas 95 รวม = D1N1 + D2N1
B7 รวม = D2N2
Diesel รวม = D1N2
B10 รวม = 0 (ไม่มีในตู้)
```

### 2️⃣ คำนวณยอดเงินจากการขายน้ำมัน

```
ยอดเงิน = ยอดขาย (ลิตร) × ราคาน้ำมัน (บาท/ลิตร)

ตัวอย่าง:
Gas 95 = ลิตรรวม × ราคา 95
B7 = ลิตร B7 × ราคา B7
Diesel = ลิตร Diesel × ราคา Diesel

ยอดเงินรวมจากน้ำมัน = Gas 95 + B7 + B10 + Diesel
```

### 3️⃣ คำนวณยอดเงินสดจากตู้จ่าย

```
ยอดเงินต่อหัว = ยอดเงินสิ้นสุด - ยอดเงินเริ่มต้น

ตัวอย่าง:
D1N1 Cash = เงินสิ้นสุด - เงินเริ่มต้น
D1N2 Cash = เงินสิ้นสุด - เงินเริ่มต้น
D2N1 Cash = เงินสิ้นสุด - เงินเริ่มต้น
D2N2 Cash = เงินสิ้นสุด - เงินเริ่มต้น

รวมตู้ 1 = D1N1 Cash + D1N2 Cash
รวมตู้ 2 = D2N1 Cash + D2N2 Cash
ยอดเงินสดรวม = รวมตู้ 1 + รวมตู้ 2
```

### 4️⃣ คำนวณยอดสุทธิ

```
ยอดสุทธิ = เงินสดที่นับได้ + 2T + เงินทุน + เงินโอน + อื่นๆ

หรือ:
totalAmount = actualCashCounted + items.twoT + items.capital + items.transfer + items.others
```

**หมายเหตุ:** ยอดสุทธินี้ไม่รวมยอดเงินคำนวณจากมิเตอร์ (`totalFuelAmount`) หรือยอดเงินจากตู้ (`cashAmount`) เพราะเป็นตัวเลขที่ใช้เปรียบเทียบกันเอง

### 5️⃣ คำนวณยอดขาด/เกิน

```
ยอดขาด/เกิน = ยอดสุทธิ - ยอดเงินสดรวมจากตู้

difference = totalAmount - cashAmount

ถ้า difference > 0: เกิน (Over) — เงินที่นับได้ + รายการอื่นๆ มากกว่ายอดในตู้
ถ้า difference < 0: ขาด (Short) — เงินที่นับได้ + รายการอื่นๆ น้อยกว่ายอดในตู้
ถ้า difference = 0: พอดี
```

**สิ่งที่ใช้เปรียบเทียบ:**
- `totalAmount` = เงินสดที่นับได้จริง + รายการอื่นๆ (2T, ทุน, โอน, อื่นๆ)
- `cashAmount` = ยอดเงินจากตู้จ่าย (คำนวณจาก dispenserCash ทุกหัว)
- **ไม่** ใช้ `totalFuelAmount` (ยอดเงินคำนวณจากมิเตอร์) มาเปรียบเทียบในสูตรนี้ เพราะตู้จ่ายและมิเตอร์เป็นระบบคนละชุดกัน

### 🧮 ขั้นตอนการคำนวณ:

#### ขั้นที่ 1: คำนวณยอดขาย (ลิตร)

```
ตู้ 1 หัว 1 (95): 
= max(0, 15678.25 - 15234.50)
= max(0, 443.75)
= 443.75 ลิตร

ตู้ 1 หัว 2 (Diesel):
= max(0, 8923.75 - 8750.00)
= max(0, 173.75)
= 173.75 ลิตร

ตู้ 2 หัว 1 (95):
= max(0, 22456.80 - 22150.00)
= max(0, 306.80)
= 306.80 ลิตร

ตู้ 2 หัว 2 (B7):
= max(0, 12567.50 - 12345.00)
= max(0, 222.50)
= 222.50 ลิตร
```

**สรุปยอดขายต่อประเภท:**
| ประเภท | ลิตร |
|--------|------|
| Gas 95 | 443.75 + 306.80 = **750.55** |
| B7 | **222.50** |
| Diesel | **173.75** |
| B10 | **0** |
| **รวม** | **1,146.80** |

---

#### ขั้นที่ 2: คำนวณยอดเงินจากการขายน้ำมัน

```
Gas 95 = 750.55 × 49.35 = 37,039.64 บาท
B7 = 222.50 × 37.45 = 8,332.63 บาท
Diesel = 173.75 × 32.99 = 5,732.01 บาท
B10 = 0 × 36.95 = 0 บาท

ยอดเงินรวมจากน้ำมัน = 37,039.64 + 8,332.63 + 5,732.01 + 0
                   = 51,104.28 บาท
```

**สรุปยอดเงินต่อประเภท:**
| ประเภท | ลิตร | ราคา/ลิตร | ยอดเงิน (บาท) |
|--------|------|-----------|---------------|
| Gas 95 | 750.55 | 49.35 | 37,039.64 |
| B7 | 222.50 | 37.45 | 8,332.63 |
| Diesel | 173.75 | 32.99 | 5,732.01 |
| B10 | 0 | 36.95 | 0 |
| **รวม** | **1,146.80** | - | **51,104.28** |

---

#### ขั้นที่ 3: คำนวณยอดเงินสดจากตู้จ่าย

```
ตู้ 1 หัว 1 (95) = 26,891.18 - 5,000 = 21,891.18 บาท
ตู้ 1 หัว 2 (Diesel) = 8,572.90 - 3,000 = 5,572.90 บาท
รวมตู้ 1 = 21,891.18 + 5,572.90 = 27,464.08 บาท

ตู้ 2 หัว 1 (95) = 18,848.31 - 4,000 = 14,848.31 บาท
ตู้ 2 หัว 2 (B7) = 10,835.41 - 2,500 = 8,335.41 บาท
รวมตู้ 2 = 14,848.31 + 8,335.41 = 23,183.72 บาท

ยอดเงินสดรวม = 27,464.08 + 23,183.72 = 50,647.80 บาท
```

**สรุปยอดเงินสด:**
| ตู้ | หัว | ประเภท | ยอดเงิน (บาท) |
|-----|-----|--------|---------------|
| 1 | 1 | 95 | 21,891.18 |
| 1 | 2 | Diesel | 5,572.90 |
| | | **รวมตู้ 1** | **27,464.08** |
| 2 | 1 | 95 | 14,848.31 |
| 2 | 2 | B7 | 8,335.41 |
| | | **รวมตู้ 2** | **23,183.72** |
| | | **ยอดเงินสดรวม** | **50,647.80** |

#### ขั้นที่ 4: คำนวณยอดสุทธิ

```
ยอดสุทธิ = เงินสดที่นับได้ + 2T + เงินทุน + เงินโอน + อื่นๆ

#### ขั้นที่ 5: คำนวณยอดขาด/เกิน

```

## การทำงานแบบ Offline-First (ไม่ต้องต่อเน็ต)

ระบบรองรับการทำงานแบบ **Offline-First** ใช้งานได้แม้ไม่มีอินเทอร์เน็ต

### สถานะการเชื่อมต่อ
- **🟢 Online**: เชื่อมต่ออินเทอร์เน็ตปกติ ข้อมูลซิงค์แบบ Real-time
- **🟡 Offline Mode**: ไม่มีเน็ต แต่ยังใช้งานได้ ข้อมูลจะถูกเก็บในเครื่อง
- **🔴 Offline**: ไม่มีเน็ต ระบบแสดง banner แจ้งเตือน

### ฟีเจอร์ที่รองรับ Offline

| ฟีเจอร์ | ใช้งาน Offline | Sync เมื่อ Online |
|---------|----------------|-------------------|
| POS (ขายสินค้า) | ✅ ได้ | ✅ อัตโนมัติ |
| ลงเวลา | ✅ ได้ | ✅ อัตโนมัติ |
| ขอลา | ✅ ได้ | ✅ อัตโนมัติ |
| บัญชีรายวัน | ✅ ได้ | ✅ อัตโนมัติ |
| ดูตารางงาน | ✅ ใช้ข้อมูลล่าสุด | - |
| ดูสินค้า | ✅ ใช้ข้อมูลล่าสุด | - |

### การใช้งานในโหมด Offline

1. **เมื่อเน็ตขาด**: ระบบแสดง banner แดง "You are offline"
2. **ทำงานตามปกติ**: ขายสินค้า ลงเวลา ยื่นลา ได้เหมือนเดิม
3. **ข้อมูลถูกเก็บ**: บันทึกลง IndexedDB ใน browser
4. **เมื่อกลับมาออนไลน์**: ระบบซิงค์ข้อมูลอัตโนมัติ

### การตรวจสอบสถานะซิงค์

- ดูที่ **Offline Banner** ด้านบนหน้าจอ
- แสดงจำนวนรายการที่รอซิงค์ (pending)
- กดปุ่ม "Sync Now" เพื่อซิงค์ทันที (เมื่อออนไลน์)
- กด "Details" เพื่อดูสถิติการใช้งาน

### ข้อควรระวัง

- **อย่าปิด browser** ถ้ามีรายการรอซิงค์ (อาจสูญหาย)
- **แนะนำ**: เปิดหน้าเว็บทิ้งไว้จนกว่าจะซิงค์สำเร็จ
- **รีเฟรชหน้า**: ถ้าซิงค์ไม่สำเร็จ ลองรีเฟรชหน้าเว็บ

## การใช้งานแต่ละฟีเจอร์

### 1. การจัดการสาขา (Branch/Station Management)

> **สิทธิ์:** Admin เท่านั้น (Manager/Staff เห็นแต่ไม่สามารถจัดการได้)

#### การสร้างสาขาใหม่
1. ไปที่เมนู "สาขา" (`/stations`)
2. คลิก "เพิ่มสาขา"
3. กรอกข้อมูล:

### 5. ระบบบัญชีรายวัน (Daily Accounting)

#### ขั้นตอนบันทึกบัญชี
1. ไปที่เมนู "บัญชีรายวัน"
2. คลิก "บันทึกบัญชี"
3. เลือกวันที่และกะ
4. ระบบแสดงพนักงานในกะนั้น
5. กรอกยอดมิเตอร์เริ่มต้นและสิ้นสุด (4 หัวจ่าย)
6. กรอกยอดเงินในแต่ละตู้ (เริ่มต้น-สิ้นสุด)
7. กรอกรายการอื่นๆ (2T, เงินทุน, เงินโอน, อื่นๆ)
8. ระบบคำนวณยอดเงินอัตโนมัติ
9. ตรวจสอบยอดขาด/เกิน
10. กด "บันทึก"

#### ฟีเจอร์พิเศษ
- **Auto-fill ค่าเริ่มต้น**: เมื่อเปิด dialog บันทึกใหม่ ระบบจะดึงค่า "สิ้นสุด" ของรายการล่าสุดมาใส่เป็น "เริ่มต้น" โดยอัตโนมัติ (ทั้งยอดลิตรและยอดเงิน)
- **ล็อกค่าเริ่มต้น**: ฟิลด์ "ยอดเงินเริ่มต้น" และ "ยอดลิตรเริ่มต้น" จะถูกปิดไม่ให้แก้ไขได้ (ทั้งตอนเพิ่มใหม่และแก้ไข) เพราะเป็นค่าสะสมจากรอบก่อนหน้า
- **ตั้งราคาน้ำมัน**: ปรับราคาน้ำมันได้ตามวันที่มีผล โดยระบบจะดึงราคาล่าสุดที่บันทึกไว้มาแสดงอัตโนมัติ
- **คำนวณอัตโนมัติ**: คำนวณยอดเงินและยอดขาด/เกินทันที
- **การคำนวณที่ถูกต้อง**: ยอดเงินสดรวม (`cashAmount`) คำนวณจาก `dispenserCash` โดยตรง (`สิ้นสุด - เริ่มต้น` ทุกหัว) ไม่ใช่ค่าที่ส่งมาจากฟอร์ม ทำให้ยอดขาด/เกินตรงเสมอ

#### การสำรองและกู้คืนข้อมูล (Backup & Restore)

1. ไปที่ "ตั้งค่า" → "สำรองและกู้คืนข้อมูล"
2. คลิกปุ่ม "สำรองข้อมูล"

4. คลิก "สำรอง"
5. ระบบจะดาวน์โหลดไฟล์ JSON อัตโนมัติ
6. ไฟล์จะมีชื่อรูปแบบ: `backup-gasstation-YYYY-MM-DDTHH-mm-ss.json`

| **คลัง** | fuel_inventory | สต็อกน้ำมัน |

**การกู้คืนข้อมูล (Restore):**
1. ไปที่ "ตั้งค่า" → "สำรองและกู้คืนข้อมูล"
2. คลิกปุ่ม "กู้คืนข้อมูล"
3. เลือกไฟล์สำรอง (.json) จากเครื่อง
4. ตรวจสอบรายละเอียดไฟล์:
   - วันที่สำรอง
   - ผู้สำรอง
   - เวอร์ชัน
5. เลือกตารางที่ต้องการกู้คืน (แสดงเฉพาะตารางที่มีในไฟล์):
   - แสดงจำนวนรายการในแต่ละตาราง
   - ใช้ปุ่ม "เลือกทั้งหมด" หรือ "ยกเลิกทั้งหมด"
   - หรือเลือก/ยกเลิกแต่ละตารางเอง
6. คลิก "กู้คืน"
7. รอระบบดำเนินการ
8. **แนะนำ:** รีเฟรชหน้าเว็บเพื่อโหลดข้อมูลใหม่

**⚠️ คำเตือน:**
- การกู้คืนข้อมูลจะเขียนทับข้อมูลที่มีอยู่ (upsert)
- ระบบจะจัดเรียงลำดับการกู้คืนตามความสัมพันธ์ของตารางอัตโนมัติ
- แนะนำให้สำรองข้อมูลปัจจุบันก่อนการกู้คืนเสมอ
- ใช้ไฟล์สำรองที่สร้างจากระบบนี้เท่านั้น

**ตัวอย่างการใช้งาน:**
- **สำรองเฉพาะพนักงาน**: เลือก users, profiles, positions, skills
- **สำรองเฉพาะบัญชี**: เลือก daily_accounting, fuel_prices
- **ย้ายข้อมูลไปเซิร์ฟเวอร์ใหม่**: เลือกทั้งหมด แล้วกู้คืนทั้งหมด

**การสำรองฐานข้อมูลระดับ DB (Database-Level Backup):**
สิทธิ์: **Admin เท่านั้น**

1. ไปที่ "ตั้งค่า" → "สำรองและกู้คืนข้อมูล"
2. คลิกปุ่ม "สำรองฐานข้อมูล (pg_dump)"
3. ระบบจะสำรองข้อมูล 22 ตารางหลักเป็นไฟล์ SQL (`TRUNCATE` + `INSERT`)
4. ไฟล์จะถูกอัปโหลดไปยัง Supabase Storage bucket `backups` โดยอัตโนมัติ
5. Admin/Manager ทุกคนจะได้รับการแจ้งเตือนแบบ Real-time เมื่อสำรองสำเร็จ
6. ดาวน์โหลดไฟล์ได้จาก Supabase Dashboard → Storage → `backups`

**ตั้งค่าสำรองอัตโนมัติด้วย cron-job.org:**
1. สมัครและเข้าสู่ระบบที่ https://cron-job.org
2. สร้าง cron job ใหม่:
   - **URL:** `https://your-project.supabase.co/functions/v1/backup-database`
   - **Method:** `POST`
   - **Headers:**
     - `Content-Type: application/json`
     - `x-backup-secret: your-random-secret-key`
   - **Body:** ว่างหรือ `{}`
3. กำหนดตารางเวลา เช่น 02:00 น. ทุกวัน (ตามเวลาไทย)
4. ทุกครั้งที่สำรองสำเร็จ Admin/Manager จะได้รับการแจ้งเตือนในแอป

#### การขายน้ำมัน

2. เลือกโหมดการขาย:
   - **ขายตามเงิน (บาท)** (ค่าเริ่มต้น): ใส่จำนวนเงิน ระบบคำนวณลิตรให้
   - **ขายตามลิตร**: ใส่จำนวนลิตร ระบบคำนวณเงินให้
3. กด "Add to Cart"

#### การขายสินค้า
1. กดปุ่มสินค้าที่ต้องการ (แสดงเป็น Grid)
2. ระบุจำนวน
3. กด "Add to Cart"
4. **สแกนบาร์โค้ด**: ใช้ scanner หรือพิมพ์รหัสบาร์โค้ด

#### การชำระเงิน
1. กดปุ่ม "Pay" (สีเขียว)
2. เลือกวิธีชำระเงิน:
   - **เงินสด (Cash)**: ใส่จำนวนเงินที่รับมา ระบบคำนวณเงินทอน
   - **เครดิตการ์ด (Credit Card)**: บันทึกหมายเลขอ้างอิง (optional)
   - **QR Code**: รองรับ PromptPay, Thai QR
   - **E-Wallet**: บันทึกหมายเลขอ้างอิง
3. ใส่จำนวนเงิน
4. กด "Add Payment"
5. สามารถใช้หลายวิธีชำระเงินได้ (เช่น เงินสด + QR Code)
6. ถ้าชำระครบ กด "Complete Sale"

#### ฟีเจอร์พิเศษ
- **ตะกร้าสินค้า**: แสดงรายการทั้งหมด พร้อมยอดรวม
- **ส่วนลด**: เพิ่มส่วนลดเป็นเปอร์เซ็นต์หรือจำนวนเงิน
- **ภาษี**: คำนวณ VAT 7% อัตโนมัติ
  - น้ำมัน: ราคารวม VAT แล้ว (VAT 0% ในระบบ)
  - สินค้า/บริการ: คำนวณ VAT 7% เพิ่ม
- **เงินทอน**: คำนวณอัตโนมัติ
- **ตัดสต็อก**: ตัดสต็อกสินค้าอัตโนมัติเมื่อขายสำเร็จ
- **ยกเลิกการขาย**: กด "Cancel" พร้อมระบุเหตุผล
- **ใบเสร็จ**: แสดงใบเสร็จบนหน้าจอ พร้อม QR Code

#### การขายตามเงิน (บาท) - แนะนำ
เมื่อเลือกขายตามเงิน:
- ใส่จำนวนเงินที่ลูกค้าต้องการ (เช่น 500 บาท)
- ระบบคำนวณลิตร = เงิน ÷ ราคาต่อลิตร
- ยอดเงินในตะกร้าแสดงตามที่กรอก (ไม่มีปัดเศษ)
- **เหมาะสำหรับ**: ลูกค้าต้องการเติมตามงบประมาณ (เช่น "เติมให้เต็ม 500")

#### การทำงานแบบ Offline
- POS ทำงานได้ปกติแม้ไม่มีเน็ต
- ข้อมูลการขายถูกเก็บในเครื่อง
- เมื่อกลับมาออนไลน์ ระบบจะซิงค์ข้อมูลและตัดสต็อกอัตโนมัติ
- สามารถขายต่อเนื่องได้โดยไม่สะดุด

### 10. ระบบประวัติการเปลี่ยนแปลง (Audit Logs)

> **สิทธิ์:** Admin, Manager เท่านั้น

#### การเข้าถึง
- ผ่านเมนู **"ประวัติการเปลี่ยนแปลง"** ใน Sidebar (อยู่ในกลุ่ม "ตั้งค่าระบบ")
- หรือคลิกที่ card **"ประวัติการเปลี่ยนแปลง"** ในหน้า **ตั้งค่า**

#### ฟีเจอร์หลัก
- **ค้นหา** - ค้นหาจาก Record ID, ผู้ดำเนินการ, หรืออีเมล
- **กรองข้อมูล** - กรองตามตาราง (เช่น พนักงาน, บัญชีรายวัน, การขาย), การกระทำ (สร้าง/แก้ไข/ลบ), และช่วงวันที่
- **สถิติสรุป** - แสดงจำนวนรายการทั้งหมด แยกตามการกระทำ (สร้าง/แก้ไข/ลบ)
- **ชื่อรายการที่อ่านเข้าใจได้** - แทนที่จะแสดง UUID ที่อ่านไม่ออก ระบบจะดึงชื่อ/วันที่/ข้อมูลสำคัญจากข้อมูลที่บันทึกไว้ เช่น:
  - พนักงาน → แสดงชื่อจริง (สมชาย ใจดี)
  - สินค้า/สาขา → แสดงชื่อ
  - ตารางงาน/บัญชี → แสดงวันที่
  - คำขอลา → แสดงเหตุผล
- **ผู้ดำเนินการที่ชัดเจน** - แสดงชื่อผู้ใช้งานจากโปรไฟล์ (ไม่ใช่แค่ "system"):
  - ผู้ใช้ที่ login → แสดงชื่อเต็มจาก profile
  - ระบบอัตโนมัติ → แสดง "ระบบอัตโนมัติ" พร้อมไอคอน
  - มีแต่อีเมล → แสดงอีเมล
- **ดูรายละเอียด (Diff View)** - คลิกที่แถวเพื่อดูรายละเอียด:
  - **สร้าง** - แสดงข้อมูลที่สร้าง (JSON)
  - **แก้ไข** - แสดงเฉพาะฟิลด์ที่เปลี่ยนแปลง พร้อมค่าก่อนและหลังแก้ไข
  - **ลบ** - แสดงข้อมูลก่อนลบ (JSON)
- **Pagination** - เลือกดูหน้าถัดไป/ก่อนหน้า เลือกจำนวนรายการต่อหน้า (10/25/50/100)
- **Export CSV** - ดาวน์โหลดผลลัพธ์ที่กรองไว้เป็นไฟล์ CSV (รองรับภาษาไทยใน Excel)

#### การใช้งาน
1. ไปที่เมนู **"ประวัติการเปลี่ยนแปลง"**
2. ใช้ช่อง **ค้นหา** หรือ **ตัวกรอง** เพื่อหารายการที่ต้องการ
3. คลิกที่แถวเพื่อดูรายละเอียดการเปลี่ยนแปลง
4. กด **Export CSV** เพื่อดาวน์โหลดรายงาน

## 🔄 Real-time Subscriptions (Supabase Realtime)

ระบบรองรับการอัปเดตข้อมูลแบบ **Real-time** ผ่าน Supabase Realtime โดยไม่ต้องรีเฟรชหน้าเว็บ

### ตารางที่รองรับ Real-time
ทุกตารางในระบบจะอัปเดตแบบ Real-time โดยอัตโนมัติ:
- **ตารางกะ**: `schedules`, `shifts`, `leave_requests`, `swap_requests`
- **พนักงาน**: `profiles`, `users`, `positions`, `skills`
- **ลงเวลา**: `attendances`
- **บัญชี**: `daily_accounting`, `fuel_prices`
- **คลังสินค้า**: `fuel_inventory`, `fuel_deliveries`, `products`, `suppliers`
- **การขาย**: `sales`
- **เงินเดือน**: `payroll_periods`, `payroll_records`
- **แจ้งเตือน**: `notifications`, `alerts`
- **Audit**: `audit_logs`
- **สมาชิก**: `customers`

### การทำงาน
1. เมื่อผู้ใช้คนใดคนหนึ่งเพิ่ม/แก้ไข/ลบข้อมูล → ระบบจะส่ง event ไปยังทุกคนที่เปิดหน้านั้นอยู่
2. ข้อมูลจะถูก reload อัตโนมัติ (มี debounce 800ms ป้องกันการกระพริบ)
3. Cache ในเครื่องจะถูกเคลียร์และโหลดข้อมูลใหม่จาก Supabase
4. แสดง Toast แจ้งเตือนเมื่อมี notification ใหม่เข้ามา

### ข้อควรระวัง
- ต้องเปิดใช้งาน **Realtime** ใน Supabase Dashboard (Database → Replication)
- ถ้าไม่เห็นข้อมูลอัปเดต → ลองรีเฟรชหน้า 1 ครั้ง
- การแก้ไขข้อมูลพร้อมกันหลายคน → ใช้หลักการ "Last Write Wins"

---

## หมายเหตุสำคัญ

### ⚠️ ข้อควรระวังในการคำนวณ

1. **ค่าติดลบ**: ระบบจะตั้งค่าเป็น 0 ถ้าผลลบ (ใช้ `max(0, value)`)
2. **ทศนิยม**: ปัดเศษทศนิยม 2 ตำแหน่ง
3. **ข้อมูลเก่า**: รองรับ backward compatibility สำหรับข้อมูลที่ไม่มี dispenser2
4. **ราคาน้ำมัน**: ต้องตั้งราคาก่อนบันทึกบัญชี

### 🔧 การแก้ไขข้อผิดพลาด

| ปัญหา | วิธีแก้ไข |
|-------|-----------|
| ยอดเงินคำนวณไม่ตรง | ตรวจสอบราคาน้ำมัน |
| ไม่พบพนักงานในกะ | ตรวจสอบตารางงาน |
| มิเตอร์ไม่แสดงค่าเดิม | ตรวจสอบบัญชีล่าสุด |
| ยอดขาด/เกินผิดปกติ | ตรวจสอบยอดเงินเริ่มต้น-สิ้นสุด |
| POS ซิงค์ไม่สำเร็จ | ตรวจสอบการเชื่อมต่อ กด Sync Now |
| ข้อมูล offline หาย | อย่าปิด browser ถ้ามีรายการรอซิงค์ |
| สต็อกไม่ตัด | ตรวจสอบว่าขาย completed และมีเน็ต |
| ไม่สามารถขายได้ | ตรวจสอบสต็อกสินค้า (ถ้าหมดจะขายไม่ได้) |
| โปรโมชั่นไม่แสดง | ตรวจสอบว่าเปิดใช้งาน และอยู่ในช่วงเวลา/วันที่ที่กำหนด |
| ยอดขาด/เกินไม่ตรง | ตรวจสอบว่า `totalAmount` คำนวณจาก `actualCashCounted + items` ไม่ใช่รวม `totalFuelAmount` |
